# IDC Valuation Overlay

Use this guide when valuing an IDC or AI-compute transition story.

## Source Map

| Source | What It Supports | URL |

## Method Hints

- Use SOTP when the company still has legacy engineering or manufacturing assets plus new IDC assets.
- Use EV/EBITDA carefully because heavy capex can make EBITDA look better than true owner economics.
- Use asset-based or replacement-cost framing only when there is a realistic path to monetization and power access.
- Use DCF only if utilization, ramp-up, and financing assumptions are explicitly disclosed or can be reasonably bounded.

## Inputs to Force

- Asset-loading or utilization path
- Segment mix between one-off delivery and recurring operation
- Debt cost and maturity profile
- Additional capex still required before steady-state operation
- Probability of project delay, contract revision, or slower ramp

## Output Reminder

Show which valuation upside depends on execution, which depends on financing relief, and which depends on real recurring monetization.
